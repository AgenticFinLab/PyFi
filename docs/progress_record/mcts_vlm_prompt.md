# 针对结果问题的提示词


# 基于金融趋势图像的环环相扣问答对基准构建综合规范

## 一、背景与目标
构建大规模、高质量的金融趋势图像问答对基准数据集，用于系统评估视觉理解模型在金融场景下对线条类图像的多维度识别、理解、分析及判断能力。  

核心设计思路为：借鉴蒙特卡洛树搜索思想，通过构建"环环相扣的问答链条"模拟"思维链推理"过程——以人工设计的"最终问题"答案正确性为核心依据，反向量化筛选链条中各节点问答对的质量，最终形成可用于模型能力评估的标准化基准。  

该基准的核心价值在于：通过链条式推理结构，实现对模型从基础视觉识别到高阶决策支持的全能力链评估，且以"最终问题"的"终局正确性"（类似围棋胜负判定）为标准，回溯验证各节点的有效性，确保数据集的高质量与难度梯度合理性。


## 二、核心定义与范围
### （一）金融趋势图像类型
输入数据为以线条为核心信息载体的金融类图像，涵盖但不限于：  
- 交易类图表：股票/期货K线图（含日/周/月线、均线、成交量线等衍生线条）、期货/期权走势图、汇率波动线图等；  
- 业务分析类图表：金融业务发展趋势线图（如用户增长曲线、营收趋势线）、业绩对比分析图（如不同产品线营收折线对比）、指标波动示意图（如波动率曲线、相关性折线）等；  
- 分析类线图：风险指标趋势线、流动性监测曲线、资产配置比例变化折线等；  
- 流程示意类图表：金融交易流程图、资金流转示意图、风险传导逻辑线图、业务审批线框图等；  
- 其他含线条信息的金融专业图像（如监管指标合规趋势线、客户分层迁徙路径图）。  


### （二）问答对链条
由N（N≥3）个节点组成的逻辑链条，每个节点为一个独立问答对，前序节点的答案为后续节点的推理基础，最终共同指向"最终问题"的答案。  

链条需满足"递进式逻辑"：从"数据提取→计算分析→特征识别→趋势判断→决策推导"逐步深入，禁止跳跃性或无关性推理，确保逻辑连贯性。  


### （三）最终问题
链条的终端问题，是评估整个推理过程有效性的核心依据。其设计需满足特定规范（详见第六部分），且答案的正确性将作为回溯评估链条中所有节点有效性的唯一标准（类似"终局胜负决定每步价值"）。  


## 三、问答对链条设计总原则
1. **逻辑连贯性**：后续节点问答对必须严格基于前序节点的结论展开，形成无断层的推理路径，禁止出现跳跃性或无关性问题；  
2. **客观性**：所有节点的问题均需为客观题（单选题、多选题、判断题），答案唯一且无争议；最终问题答案需客观可验证，不受主观解读影响；  
3. **能力覆盖性**：链条需系统覆盖视觉理解模型在金融场景下的多元能力（详见第五部分），且各节点能力考察需呈现梯度化（从基础到高级）；  
4. **可评估性**：以最终问题答案的正确性为唯一标准，反向为链条中所有节点赋分（正确则节点有效，错误则节点无效），模拟围棋"终局胜负决定每步价值"的评估逻辑。  


## 四、节点问答对设计规范
### （一）节点构成要素
每个节点必须包含以下字段，确保信息完整与指向明确： 
- **父节点编号(parent)**: 父节点，如本节点是根节点，则父节点为0；
- **本节点编号(node)**: 
- **问题（Question）**：针对图像内容的客观查询，需明确、无歧义；  
- **答案（Answer）**：唯一且无争议的标准答案，格式符合问题类型要求；  
- **指令（Instruction）**：对问题背景的补充说明，包括但不限于：  
  - 图像区域指引（如"针对图中2023年Q1至Q3的蓝色折线"）；  
  - 术语解释（如"MACD线指图中橙色曲线"）；  
  - 推理范围限定（如"仅基于图中2022年数据"）；  
- **考察能力类别（Capability）**：明确该问题所评估的模型能力维度（详见第五部分）。  


### （二）问题类型与格式
1. **单选题**：题干明确提出一个问题，提供A、B、C、D四个选项（唯一正确答案）。  
   格式示例："图中2023年Q3的营收趋势线斜率为？A.正 B.负 C.零 D.无法判断"。  

2. **多选题**：题干明确提出一个问题，提供A、B、C、D四个选项（多个正确答案）。  
   格式示例："图中属于均线的线条有？A.线1（红色实线） B.线2（蓝色虚线） C.线3（绿色实线） D.线4（黑色点线）"。  

3. **判断题**：题干为一个基于图像的陈述，要求判断真伪。  
   格式示例："图中股票在2023年12月的收盘价高于10月收盘价（T/F）"。  


### （三）答案格式标准
- 单选题答案：直接输出选项字母（如"A"）；  
- 多选题答案：选项字母用英文逗号分隔（如"B,D"）；  
- 判断题答案：输出"T"（正确）或"F"（错误）。  


## 五、考察能力维度定义
各节点问题需明确标注考察的能力类别，且按链条递进顺序从基础到高级分布，包括但不限于：  

| 能力类别         | 核心定义                                                                 | 示例                                                                 |
|------------------|--------------------------------------------------------------------------|----------------------------------------------------------------------|
| 视觉识别能力     | 识别线条的视觉属性及基础要素，包括线条类型（如实线/虚线/点线）、颜色、样式、位置、长度、坐标刻度解读，以及线条与其他元素（如坐标轴、图例）的对应关系。 | 区分K线实体与均线、识别"红色线条代表上涨"、解读纵轴刻度单位（如"元/股"）。 |
| 数据提取能力     | 从线条中提取具体数值、时间节点等基础信息，包括特定时间点数值读取（如K线的开盘价、收盘价）、区间极值识别（如某季度营收最大值）、线条交叉点信息提取（如两条均线的交叉日期）、时间节点（如趋势线的拐点日期）等。 | 提取"2023年6月15日的收盘价"、识别"2022年Q2的营收峰值"。               |
| 计算分析能力     | 基于提取的数据进行计算、比较或基础分析，包括涨跌幅计算（如两日收盘价涨跌幅）、均线值计算、不同时间段线条数值对比（如Q1与Q2营收差值）、斜率计算等。 | 计算"2023年5月的月涨跌幅"、比较"线A与线B在3月的数值大小"。           |
| 趋势分析能力     | 识别线条的特征模式与趋势规律，包括涨/跌趋势判断（如连续3日收盘价上涨）、斜率变化解读（如趋势线从平缓变陡峭）、周期性规律识别（如季度性波动）、特征形态识别（如K线的"头肩顶"形态、趋势线的"陡峭/平缓"特征）等。 | 判断"2023年Q3营收呈上升趋势"、识别"价格线形成双顶形态"。               |
| 逻辑推理能力     | 基于前序结论推导线条间的关系或隐含信息，包括多线条关系推导（如成交量线与价格线的相关性）、因果关系判断（如"成交量上升是否导致价格上涨"）、隐含信息挖掘（如"趋势线拐点是否预示政策影响"）、短期趋势预测等。 | 推导"成交量放大与价格上涨的关联性"、预测"未来1周的价格趋势"。         |
| 决策支持能力     | 综合前序结论给出符合金融逻辑的决策建议，包括投资建议（如"是否买入"）、风险预警（如"某业务指标突破风险阈值"）、业务优化方向判断（如"用户增长放缓的应对策略"）等。 | 建议"基于当前趋势增持某股票"、预警"流动性指标趋势预示潜在风险"。     |

问题的复杂性：complexity，1-10分



## 六、最终问题设计规范
最终问题作为链条有效性的评估终点，需同时满足以下要求：  

1. **可判断性**：答案无需专业金融知识即可被普通大学生（无金融背景）准确判断，且判断过程不超过30秒。  
   示例："以下哪项符合图中最终趋势结论？A.上涨 B.下跌 C.持平"。  

2. **依赖性**：必须依赖链条中所有前序节点的结论才能推导，无法通过直接观察图像表面信息或常识回答。即模型若跳过前序推理，仅通过图像颜色深浅、线条走向等表面特征无法得出正确答案。  
   示例：需基于前序节点计算的"3个季度的涨跌幅"和"趋势斜率变化"，才能回答"该业务是否达成年度增长目标"，无法直接通过"线条向上"判断。  

3. **客观性**：答案唯一且无争议，选项需互斥（单选题）或明确（多选题），**禁止出现模糊选项（如“无法判断”“不确定”“可能”等）**，确保答案的确定性。  

4. **区分性**：能有效区分模型与人类的推理能力——人类通过逐步推导可得出答案，而模型若存在前序推理漏洞则必然错误，以此暴露模型在复杂逻辑链中的缺陷。  

5. **关联性**：与前序节点的最终推理结论直接相关，且紧密结合金融场景核心领域（如投资决策、风险判断、业务预测等）。  
   示例：前序节点最终推导出"2023年Q4价格线较Q1上涨20%且趋势持续"，最终问题可设计为"基于全年趋势，该股票的年度表现为？A.上涨 B.下跌 C.持平"。  

6. **难度适配性**：其难度需与链条长度及前序节点的梯度设计相匹配（如5节点链条的最终问题难度应高于3节点链条）。  



## 七、问答对链条构建规范
### （一）总原则
链条需满足"递进式逻辑"与"能力梯度"，具体包括：  
1. **依赖性**：后续节点的问题必须基于前序节点的答案展开，形成严格的逻辑依赖（如节点2的问题无法脱离节点1的答案独立解答）；  
2. **完整性**：链条节点覆盖"最终问题"所需的全部推理环节，无逻辑断层；  
3. **梯度性**：从基础能力（如视觉识别）逐步过渡到高级能力（如决策支持），避免能力考察的跳跃。  


### （二）结构要求
1. **逻辑链示例**：  
   - 节点1（视觉识别）：识别线条对应的金融指标（如"红色线条是否为营收"）；  
   - 节点2（数据提取）：提取特定时间点的指标数值（如"2023年Q1营收"）；  
   - 节点3（计算分析）：计算指标的变化值（如"Q1与Q2营收差值"）；  
   - 节点4（趋势分析）：判断指标的整体趋势（如"2023年上半年营收是否增长"）；  
   - 最终问题：综合前序结论回答核心问题（如"该业务是否达成上半年增长目标？A.是 B.否"）。  

2. **长度适配**：链条长度（N）根据图像复杂度动态调整：  
   - 简单图像（如单一线条流程图）：N=3-5；  
   - 复杂图像（如多指标K线图）：N=5-8。  


## 八、质量评估标准
1. **逻辑自洽性**：链条中各节点问答对之间无矛盾，推理路径唯一，前序答案与后续问题的逻辑关联紧密（如节点2的问题必须基于节点1的答案展开）。  
2. **答案唯一性**：所有问题的标准答案均具有排他性，无歧义（如多选题的正确选项组合唯一，不存在"既A对又B对"的情况）。  
3. **依赖有效性**：任意节点的问题均无法绕开前序节点独立解答（如删除节点2后，节点3的问题无法回答）。  
4. **能力覆盖度**：链条需涵盖至少3种及以上第五部分定义的能力维度，且呈现从基础到高级的梯度。  
5. **难度区分度**：从初始节点到最终问题，难度呈单调递增趋势，避免"前难后易"或难度波动。  
6. **金融相关性**：所有问题均需紧密结合金融场景，避免与金融逻辑无关的泛化问题（如仅问"线条颜色"而不关联金融含义）。  


## 九、输出形式要求
1. 每个问答对链条需包含：N个节点问答对（含Question、Answer、Instruction、Capability字段）+1个最终问题（含选项及标准答案）。  
2. 需单独明确列出最终问题，并配套提供完整的问答对链条示例。  
3. 需说明最终问题与前序节点的逻辑依赖关系（如"最终问题的答案需基于节点2的计算结果和节点3的趋势判断"）。

## 二、生成内容要求

2. **能力梯度设计**：列出各节点计划考察的能力类别（需从“视觉识别能力/数据提取能力/计算分析能力/趋势分析能力/逻辑推理能力/决策支持能力”中选择），确保从基础到高级递进（示例：节点1-视觉识别→节点2-数据提取→节点3-计算分析→节点4-趋势分析→节点5-逻辑推理→最终问题）。

### （二）节点问答对生成（针对需新增的节点）

3. **Instruction**：

4. **Capability**：
   - 从第五部分“考察能力维度”中选择唯一对应类别，确保与问题难度匹配。

### （三）最终问题生成

2. **依赖性**：详细阐述为何必须依赖所有前序节点结论才能解答，无法通过直接观察图像表面信息得出答案；
3. **客观性**：证明答案唯一且无争议（如选项互斥，无“无法判断”等模糊选项）；
4. **区分性**：说明为何能区分模型与人类推理能力（如“人类可通过完整逻辑链推导，模型若跳过某节点则必然错误”）；
5. **关联性**：明确与前序节点最终结论的直接关联，及在金融场景中的具体体现（如投资决策/风险判断）；
6. **难度适配性**：说明与链条长度、前序节点梯度的匹配性（如“5节点链条的最终问题需综合3种能力结论，难度高于3节点链条”）。

### （四）链条逻辑验证说明
生成后需补充以下验证内容：
1. 逻辑连贯性验证：说明各节点间的依赖关系（如“节点3的问题必须基于节点2的答案，因需用节点2提取的‘Q2数值’计算差值”）；
2. 能力覆盖度说明：列出链条覆盖的能力类别（需≥3种），并说明梯度递进关系；
3. 最终问题回溯性说明：若最终问题答案错误，为何可判定前序节点中存在无效节点（举例说明可能的无效节点）。

## 三、结构逻辑验证  
1. **依赖关系**：  
   - 第4层依赖第3层（需3-1的住房增长额+3-2的消费增长额，总和为60+40=100亿元≥90亿元，故答案为“是”）；  
   - 第3层节点依赖第2层（3-1需2-1a的Q1余额320亿元和2-1b的Q4余额380亿元，380-320=60亿元；3-2需2-2a的Q1余额280亿元和2-2b的Q4余额320亿元，320-280=40亿元）；  
   - 第2层节点依赖第1层（如2-1a需1-1a1确认红色为住房贷款、1-1a2确认左侧第一个刻度为Q1，才能提取Q1余额320亿元）。  

2. **能力梯度**：从第1层“视觉识别”（基础能力）→第2层“数据提取”（中级能力）→第3层“计算分析”（高级能力）→第4层“决策支持关联”（综合能力），梯度递进无跳跃。  

3. **最终问题回溯性**：若最终问题答案错误（如应为“否”），则必为第3层计算错误（如3-1或3-2增长额算错），或第2层数据提取错误（如Q1/Q4余额读错），或第1层视觉识别错误（如误判线条属性或季度位置）。
